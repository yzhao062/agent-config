# Meta Review Venues

## Generic Fallback

If the venue is unknown, structure the meta review like this:

1. Brief summary of the paper and its main claimed contribution.
2. Overall reviewer feedback: positive, mixed, or negative.
3. Rebuttal effect if rebuttal text exists.
4. Final recommendation using the venue scale supplied by the user.
5. Ethics note if any real issue was raised.

Keep the draft concise unless the user asks for a longer rationale.

## TNNLS Meta Review

Use this structure for TNNLS:

1. Give a 1-2 sentence paper summary focused on real merits. Do not overclaim.
2. Summarize the overall reviewer feedback as positive, mixed, or negative.
3. State the recommendation using one of these options:
- Accept
- Minor Revision
- Reject & Resubmit
- Reject
4. Mention 1-2 points raised by at least two reviewers. Avoid generic lines like "the novelty is limited" unless the shared point is truly that broad and you can make it concrete.
5. Note whether any reviewer raised a real ethics concern and whether it affects the decision.

Target length: around 200-250 words, with moderate detail.

## ICML Meta Review

Use this structure for ICML-style area chair summaries:

1. Give a 1-2 sentence paper summary based on reviewer consensus.
2. Summarize whether the review set is overall positive, mixed, or negative.
3. Explain how the rebuttal changed, or failed to change, the discussion.
4. State the recommendation using one of these options:
- Reject
- Weak accept
- Accept
- Strong accept
- Nominate for outstanding paper award
5. Add any ethics note only if there is a concrete issue.

Target shape: one or two paragraphs. Keep sentences simple.

## NeurIPS Meta Review

Use this structure for NeurIPS:

1. Give a 1-2 sentence paper summary based on reviewer consensus.
2. Summarize whether the review set is overall positive, mixed, or negative.
3. Explain the rebuttal effect when relevant.
4. State the recommendation using one of these options:
- Reject
- Accept (poster)
- Accept (spotlight)
- Accept (oral)
5. Add any ethics note only if there is a concrete issue.

Target shape: one or two paragraphs. Keep sentences simple.

## If The User Supplies A Different Venue Form

- Follow the venue's explicit field order and decision scale.
- If the user gives both a venue name and a conflicting decision scale, trust the decision scale first and flag the mismatch only if it affects the draft.
- Do not manufacture consensus language just to fit a formal template.
