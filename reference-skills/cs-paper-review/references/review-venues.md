# Review Venues

## Generic Fallback

If the venue form is unknown, use this order:

1. Short summary of the paper and its intended contribution.
2. Main strengths.
3. Main weaknesses.
4. Questions for the authors.
5. Ethics note if a real issue is present or the venue explicitly asks for it.

Keep the review concrete and moderate in length unless the user asks for a shorter or longer version.

## IJCAI Direct Review

Use this structure when the user asks for an IJCAI review or provides matching fields:

1. Give a 1-2 sentence summary of the paper and its main merits. Do not overclaim.
2. Explain the main strengths with emphasis on novelty and soundness.
3. Explain the main weaknesses with emphasis on novelty, soundness, clarity, and significance.
4. List questions for the author-feedback period only when:
- the answers could directly change the evaluation
- the answers do not require new results
5. Add an ethics note only if there is a concrete issue to flag.

Useful style choices:

- Prefer moderate detail over long essays.
- Make the weaknesses specific enough that the authors can respond directly.
- Avoid generic phrases like "incremental novelty" unless you name what the paper is actually close to.

## If The User Supplies A Different Venue Form

- Follow the exact field names and order from the user.
- If the venue requires scores that were not provided, ask for them or leave the numeric slot out of the prose draft.
- If the venue asks for confidential comments to chairs or editors, keep them separate from the author-visible review.
